# 项目开发说明

当前仓库已经按第一步开发方案拆分为前后端工作区：

- `frontend`：`Next.js + React + TypeScript`
- `backend`：`NestJS + TypeScript`

## 根目录命令

```bash
npm install
npm run dev:frontend
npm run dev:backend
```

## 环境变量

根目录提供公共示例：

- `.env.example`

子项目提供独立示例：

- `frontend/.env.example`
- `backend/.env.example`

## 下一步

按分步开发计划，下一阶段应进入第十二步：测试、部署与上线准备。

## 第二步与第三步验证命令

```bash
npm install
npm run prisma:generate --workspace backend
npm run prisma:migrate --workspace backend
npm run db:seed --workspace backend
npm run dev:backend
```

健康检查：

```bash
curl http://localhost:3001/api/health
```

认证接口检查：

```bash
curl "http://localhost:3001/api/auth/check-username?username=testuser"
curl "http://localhost:3001/api/auth/suggest-username?username=testuser"
curl -X POST http://localhost:3001/api/auth/register -H "Content-Type: application/json" -d "{\"username\":\"testuser\",\"password\":\"Test12345\",\"confirmPassword\":\"Test12345\"}"
curl -X POST http://localhost:3001/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"admin\",\"password\":\"Admin@123456\"}"
```

当前用户接口：


浏览器打开：


验证点：

- 注册页输入用户名后会实时检测是否重复
- 用户名重复时会出现“随机”按钮推荐用户名
- 注册成功后跳转到登录页
- 登录成功后：
  - 管理员跳转 `/admin`
  - 普通用户跳转 `/chat`
- 未登录直接访问 `/chat` 或 `/admin` 会跳转到 `/login`


第十步验证点：

- 会员用户可以正常发送消息

- 非流式发送会直接返回完整回复

- 会话上下文只取最近 5 轮问答

```text

```

第十一步验证点：

- 普通非会员进入聊天页会看到会员引导
- 非会员点击发送时会被前端阻断并提示开通会员
- 管理员进入聊天页不受会员限制
- 模型不可用、会话不存在、模型调用失败时页面会显示明确错误提示
- accessToken 失效后，前端会清空登录态并跳转 `/login`
